Introduction à JavaScript

JavaScript est un langage de programmation de script principalement utilisé dans les pages web interactives. C'est un langage orienté objet à prototype, multi-paradigme et dynamique.

Note : JavaScript n'est pas Java ! Bien que leurs noms soient similaires, ce sont des langages complètement différents avec des cas d'utilisation distincts.

Pourquoi apprendre JavaScript ?

  • Langage le plus populaire selon GitHub et Stack Overflow
  • Nécessaire pour le développement web frontend
  • Utilisable également en backend (Node.js)
  • Permet de créer des applications mobiles (React Native, Ionic)
  • Permet de créer des applications desktop (Electron)

Votre premier code JavaScript

Pour exécuter du JavaScript dans une page web, vous pouvez utiliser la balise <script> :

<!DOCTYPE html>

Déclaration de Variables

Il existe trois mots-clés pour déclarer des variables en JavaScript :

// Ancienne méthode (portée fonction)
var ancienneVariable = "Je suis une ancienne variable";

// Nouvelle méthode (portée bloc)
let variableModifiable = "Je peux être modifiée";
variableModifiable = "Nouvelle valeur";

// Constante (portée bloc)
const constante = "Je ne peux pas être réaffectée";
// constante = "Nouvelle valeur"; // Erreur !

Bonnes pratiques : Utilisez toujours const par défaut, et let seulement quand vous savez que la valeur va changer. Évitez var dans le code moderne.

Structures de Contrôle

Les structures de contrôle permettent de déterminer le flux d'exécution de votre code en fonction de conditions ou de répéter des blocs de code.

Conditions : if...else

La structure if...else permet d'exécuter un bloc de code si une condition est vraie, et éventuellement un autre bloc sinon.

let age = 18;

if (age < 18) {
    console.log("Vous êtes mineur.");
} else if (age >= 18 && age < 65) {
    console.log("Vous êtes majeur.");
} else {
    console.log("Vous êtes senior.");
}

Switch

La structure switch permet de tester plusieurs valeurs pour une variable :

let jour = "Lundi";

switch (jour) {
    case "Lundi":
        console.log("Début de semaine");
        break;
    case "Mardi":
    case "Mercredi":
    case "Jeudi":
        console.log("Milieu de semaine");
        break;
    case "Vendredi":
        console.log("Bientôt le week-end");
        break;
    case "Samedi":
    case "Dimanche":
        console.log("Week-end");
        break;
    default:
        console.log("Jour non reconnu");
}

Note : N'oubliez pas les instructions break pour éviter le "fall-through" (exécution des cas suivants).

Boucles

JavaScript propose plusieurs types de boucles :

for

La boucle for est idéale quand vous connaissez le nombre d'itérations.

// Affiche les nombres de 0 à 4
for (let i = 0; i < 5; i++) {
    console.log(i);
}
while

La boucle while s'exécute tant qu'une condition est vraie.

// Affiche les nombres de 0 à 4
let j = 0;
while (j < 5) {
    console.log(j);
    j++;
}
do...while

La boucle do...while s'exécute au moins une fois, puis tant que la condition est vraie.

// Demande un nombre jusqu'à ce qu'il soit valide
let nombre;
do {
    nombre = prompt("Entrez un nombre entre 1 et 10");
} while (isNaN(nombre) || nombre < 1 || nombre > 10);
for...in

La boucle for...in itère sur les propriétés énumérables d'un objet.

let personne = {nom: "Alice", age: 25, ville: "Paris"};

for (let propriete in personne) {
    console.log(`${propriete}: ${personne[propriete]}`);
}

Attention : for...in n'est pas recommandé pour les tableaux, utilisez plutôt for...of ou forEach().

for...of

La boucle for...of (ES6) itère sur les éléments d'objets itérables (tableaux, chaînes, etc.).

let fruits = ["Pomme", "Banane", "Orange"];

for (let fruit of fruits) {
    console.log(fruit);
}

Instructions de Contrôle

JavaScript propose plusieurs instructions pour contrôler l'exécution des boucles :

Instruction Description Exemple
break Sort immédiatement de la boucle while(true) { break; }
continue Passe à l'itération suivante for(let i=0; i<5; i++) { if(i===2) continue; }
return Quitte la fonction courante function test() { return; }
throw Lance une exception throw new Error("Erreur");
try...catch Gère les exceptions try { riskyOp(); } catch(e) { console.log(e); }

Fonctions en JavaScript

Les fonctions sont des blocs de code réutilisables qui effectuent une tâche spécifique. Elles sont essentielles pour organiser et structurer votre code.

Déclaration de Fonctions

Il existe plusieurs façons de déclarer des fonctions en JavaScript :

Déclaration
Expression
Fléchée
Constructeur
// Déclaration de fonction (hoisted)
function direBonjour(nom) {
    return `Bonjour, ${nom}!`;
}

console.log(direBonjour("Alice")); // "Bonjour, Alice!"
// Expression de fonction (non hoisted)
const direBonjour = function(nom) {
    return `Bonjour, ${nom}!`;
};

console.log(direBonjour("Bob")); // "Bonjour, Bob!"
// Fonction fléchée (ES6, non hoisted, pas de 'this' propre)
const direBonjour = (nom) => {
    return `Bonjour, ${nom}!`;
};

// Version raccourcie pour un seul retour
const direBonjourCourt = nom => `Bonjour, ${nom}!`;

console.log(direBonjour("Charlie")); // "Bonjour, Charlie!"
// Constructeur de fonction (peu utilisé)
const direBonjour = new Function('nom', 'return `Bonjour, ${nom}!`');

console.log(direBonjour("David")); // "Bonjour, David!"

Attention : Le constructeur de fonction est lent et pose des problèmes de sécurité. À éviter dans la plupart des cas.

Paramètres et Arguments

JavaScript offre une grande flexibilité dans la gestion des paramètres :

// Paramètres par défaut (ES6)
function saluer(nom = "Invité", salutation = "Bonjour") {
    return `${salutation}, ${nom}!`;
}

console.log(saluer()); // "Bonjour, Invité!"
console.log(saluer("Alice", "Salut")); // "Salut, Alice!"
// Arguments variables (rest parameters)
function somme(...nombres) {
    return nombres.reduce((total, num) => total + num, 0);
}

console.log(somme(1, 2, 3)); // 6
console.log(somme(1, 2, 3, 4, 5)); // 15

Fonctions d'Ordre Supérieur

Les fonctions qui prennent d'autres fonctions en argument ou retournent des fonctions sont appelées fonctions d'ordre supérieur.

// Fonction qui retourne une fonction
function creerMultiplicateur(facteur) {
    return function(nombre) {
        return nombre * facteur;
    };
}

const double = creerMultiplicateur(2);
console.log(double(5)); // 10
// Fonction qui prend une fonction en argument
function appliquerOperation(tableau, operation) {
    return tableau.map(operation);
}

let nombres = [1, 2, 3];
let carres = appliquerOperation(nombres, x => x * x);
console.log(carres); // [1, 4, 9]

Fonctions Callback

Les callbacks sont des fonctions passées à d'autres fonctions pour être exécutées plus tard.

function chargerDonnees(url, callback) {
    // Simulation de chargement asynchrone
    setTimeout(() => {
        const donnees = `Données de ${url}`;
        callback(donnees);
    }, 1000);
}

chargerDonnees('https://api.example.com', (donnees) => {
    console.log(donnees); // "Données de https://api.example.com"
});

Note : Les callbacks sont fondamentaux en JavaScript, surtout pour les opérations asynchrones, mais peuvent mener à un "callback hell". Les Promises et async/await (voir plus loin) offrent des alternatives plus propres.

Closures

Une closure est une fonction qui "se souvient" de son environnement lexical même lorsqu'elle est exécutée en dehors de celui-ci.

function creerCompteur() {
    let compte = 0;
    
    return function() {
        compte++;
        return compte;
    };
}

const compteur = creerCompteur();
console.log(compteur()); // 1
console.log(compteur()); // 2
console.log(compteur()); // 3

IIFE (Immediately Invoked Function Expression)

Une fonction qui s'exécute immédiatement après sa création.

(function() {
    console.log("Cette fonction s'exécute immédiatement !");
})();

// Avec paramètres
(function(nom) {
    console.log(`Bonjour, ${nom}!`);
})("Alice");

Utilisation : Les IIFE étaient couramment utilisées pour créer une portée privée avant l'introduction des modules ES6. Elles restent utiles pour l'initialisation et la protection de variables.

Récursion

Une fonction qui s'appelle elle-même pour résoudre un problème.

// Factorielle avec récursion
function factorielle(n) {
    if (n <= 1) return 1;
    return n * factorielle(n - 1);
}

console.log(factorielle(5)); // 120

Attention : Chaque appel récursif utilise de la mémoire sur la pile d'appels. Pour les grandes profondeurs de récursion, préférez une solution itérative.

Exercice Pratique

Créez une fonction memoize qui prend une fonction coûteuse en calcul et retourne une version mémoïsée (qui cache les résultats déjà calculés).

function memoize(fn) {
    const cache = {};
    
    return function(...args) {
        const cle = JSON.stringify(args);
        
        if (cache[cle] !== undefined) {
            return cache[cle];
        }
        
        const resultat = fn.apply(this, args);
        cache[cle] = resultat;
        
        return resultat;
    };
}

// Exemple d'utilisation
function factorielle(n) {
    console.log('Calcul de factorielle', n);
    if (n <= 1) return 1;
    return n * factorielle(n - 1);
}

const factorielleMemo = memoize(factorielle);
console.log(factorielleMemo(5)); // Calcule tout
console.log(factorielleMemo(5)); // Retourne le résultat en cache

Manipulation du DOM

Le Document Object Model (DOM) est une interface de programmation pour les documents HTML et XML. Il représente la structure du document comme un arbre d'objets.

Sélection d'Éléments

Plusieurs méthodes pour sélectionner des éléments du DOM :

// Sélection unique
document.getElementById('monId'); // Retourne un élément
document.querySelector('#monId'); // Retourne le premier élément correspondant

// Sélection multiple
document.getElementsByClassName('maClasse'); // Retourne une HTMLCollection
document.getElementsByTagName('div'); // Retourne une HTMLCollection
document.querySelectorAll('.maClasse'); // Retourne une NodeList

Note : querySelector et querySelectorAll utilisent la syntaxe des sélecteurs CSS, ce qui les rend très puissants.

Tableaux et Objets

Les tableaux et objets sont des structures de données fondamentales en JavaScript pour organiser et manipuler des données.

Tableaux (Arrays)

Les tableaux sont des listes ordonnées de valeurs.

// Création de tableaux
let fruits = ['Pomme', 'Banane', 'Orange'];
let nombres = new Array(1, 2, 3);
let vide = [];
let mixte = [1, 'texte', true, {nom: 'Alice'}];

Méthodes Principales des Tableaux

Accès et Modification 
let fruits = ['Pomme', 'Banane', 'Orange'];

// Accès par index
let premier = fruits[0]; // 'Pomme'
let dernier = fruits[fruits.length - 1]; // 'Orange'

// Modification
fruits[1] = 'Kiwi'; // ['Pomme', 'Kiwi', 'Orange']

// Longueur du tableau
let taille = fruits.length; // 3
Ajout/Suppression 
let fruits = ['Pomme', 'Banane'];

// Fin du tableau
fruits.push('Orange'); // ['Pomme', 'Banane', 'Orange']
let dernier = fruits.pop(); // 'Orange', fruits = ['Pomme', 'Banane']

// Début du tableau
fruits.unshift('Kiwi'); // ['Kiwi', 'Pomme', 'Banane']
let premier = fruits.shift(); // 'Kiwi', fruits = ['Pomme', 'Banane']

// Suppression/Insertion
fruits.splice(1, 1, 'Fraise', 'Cerise'); 
// Supprime 1 élément à l'index 1 et ajoute 'Fraise', 'Cerise'
// Résultat: ['Pomme', 'Fraise', 'Cerise']
Recherche 
let nombres = [1, 2, 3, 4, 5];

// Trouver l'index
let index = nombres.indexOf(3); // 2
let dernierIndex = nombres.lastIndexOf(3); // 2

// Vérifier la présence
let existe = nombres.includes(3); // true

// Trouver le premier élément qui satisfait une condition
let premierPair = nombres.find(n => n % 2 === 0); // 2
let indexPair = nombres.findIndex(n => n % 2 === 0); // 1
Transformation 
let nombres = [1, 2, 3];

// map - transforme chaque élément
let carres = nombres.map(n => n * n); // [1, 4, 9]

// filter - filtre les éléments
let pairs = nombres.filter(n => n % 2 === 0); // [2]

// reduce - réduit le tableau à une valeur
let somme = nombres.reduce((acc, n) => acc + n, 0); // 6

// sort - trie le tableau
let trie = nombres.sort((a, b) => b - a); // [3, 2, 1]

// reverse - inverse l'ordre
let inverse = nombres.reverse(); // [3, 2, 1]
Itération 
let fruits = ['Pomme', 'Banane', 'Orange'];

// forEach - exécute une fonction pour chaque élément
fruits.forEach(fruit => console.log(fruit));

// every - vérifie si tous les éléments satisfont une condition
let tousLongs = fruits.every(f => f.length > 3); // false

// some - vérifie si au moins un élément satisfait une condition
let certainsLongs = fruits.some(f => f.length > 5); // true

Objets

Les objets sont des collections de propriétés sous forme de paires clé-valeur.

// Création d'objets
let personne = {
    nom: 'Alice',
    age: 25,
    ville: 'Paris',
    saluer: function() {
        return `Bonjour, je m'appelle ${this.nom}`;
    }
};

// Accès aux propriétés
console.log(personne.nom); // 'Alice'
console.log(personne['age']); // 25
console.log(personne.saluer()); // "Bonjour, je m'appelle Alice"

// Modification
personne.age = 26;
personne['ville'] = 'Lyon';

// Ajout
personne.metier = 'Développeuse';

// Suppression
delete personne.ville;

Méthodes Utiles pour les Objets

let personne = {nom: 'Alice', age: 25};

// Clés
let cles = Object.keys(personne); // ['nom', 'age']

// Valeurs
let valeurs = Object.values(personne); // ['Alice', 25]

// Entrées
let entrées = Object.entries(personne); // [['nom', 'Alice'], ['age', 25]]

// Fusion

Programmation Orientée Objet en JavaScript

JavaScript est un langage orienté objet basé sur des prototypes, plutôt que sur des classes comme les langages traditionnels OOP.

Constructeurs et Prototypes

Avant ES6, JavaScript utilisait des fonctions constructeurs et des prototypes pour l'OOP.

// Fonction constructeur
function Personne(nom, age) {
    this.nom = nom;
    this.age = age;
}

// Ajout de méthode via prototype
Personne.prototype.saluer = function() {
    return `Bonjour, je m'appelle ${this.nom}`;
};

// Création d'instance

console.log(alice.saluer()); // "Bonjour, je m'appelle Alice"

Classes ES6

ES6 a introduit une syntaxe de classe plus familière aux développeurs venant d'autres langages.

class Personne {
    // Constructeur
    constructor(nom, age) {
        this.nom = nom;
        this.age = age;
    }
    
    // Méthode d'instance
    saluer() {
        return `Bonjour, je m'appelle ${this.nom}`;
    }
    
    // Méthode statique
    static comparer(a, b) {
        return a.age - b.age;
    }
}

console.log(Personne.comparer(alice, bob)); // -5 (alice est plus jeune)

JavaScript permet l'héritage via la chaîne de prototypes ou avec la syntaxe extends des classes ES6.

Encapsulation

Les champs privés (proposés en ES2022) permettent une véritable encapsulation.

class CompteBancaire {
    // Champ privé (préfixé par #)
    #solde;
    
    constructor(titulaire, soldeInitial = 0) {
        this.titulaire = titulaire;
        this.#solde = soldeInitial;
    }
    
    deposer(montant) {
        if (montant > 0) {
            this.#solde += montant;
            return `Dépôt de ${montant} effectué. Nouveau solde: ${this.#solde}`;
        }
        return "Montant invalide";
    }
    
    retirer(montant) {
        if (montant > 0 && montant <= this.#solde) {
            this.#solde -= montant;
            return `Retrait de ${montant} effectué. Nouveau solde: ${this.#solde}`;
        }
        return "Montant invalide ou solde insuffisant";
    }
    
    getSolde() {
        return this.#solde;
    }
}

let compte = new CompteBancaire('Alice', 1000);
console.log(compte.deposer(500)); // "Dépôt de 500 effectué. Nouveau solde: 1500"
console.log(compte.retirer(200)); // "Retrait de 200 effectué. Nouveau solde: 1300"

  • Une classe Livre avec titre, auteur, ISBN et statut (disponible/emprunté)
  • Une classe Bibliotheque qui peut ajouter/supprimer des livres, rechercher par titre/auteur/ISBN, et gérer les emprunts/retours

class Livre {
    constructor(titre, auteur, ISBN) {
        this.titre = titre;
        this.auteur = auteur;
        this.ISBN = ISBN;
        this.disponible = true;
        this.emprunteur = null;
    }
}

class Utilisateur {
    constructor(nom, id) {
        this.nom = nom;
        this.id = id;
        this.livresEmpruntes = [];
    }
    
    emprunterLivre(livre) {
        if (livre.disponible) {
            livre.disponible = false;
            livre.emprunteur = this.id;
            this.livresEmpruntes.push(livre.ISBN);
            return `${this.nom} a emprunté "${livre.titre}"`;
        }
        return `"${livre.titre}" n'est pas disponible`;
    }
    
    retournerLivre(livre) {
        if (!livre.disponible && livre.emprunteur === this.id) {
            livre.disponible = true;
            livre.emprunteur = null;
            this.livresEmpruntes = this.livresEmpruntes.filter(isbn => isbn !== livre.ISBN);
            return `${this.nom} a retourné "${livre.titre}"`;
        }
        return `"${livre.titre}" n'a pas été emprunté par ${this.nom}`;
    }
}

class Bibliotheque {
    constructor() {
        this.livres = [];
    }
    
    ajouterLivre(livre) {
        this.livres.push(livre);
        return `"${livre.titre}" ajouté à la bibliothèque`;
    }
    
    supprimerLivre(isbn) {
        const index = this.livres.findIndex(l => l.ISBN === isbn);
        if (index !== -1) {
            const [livre] = this.livres.splice(index, 1);
            return `"${livre.titre}" supprimé de la bibliothèque`;
        }
        return "Livre non trouvé";
    }
    
    rechercherParTitre(titre) {
        return this.livres.filter(l => l.titre.includes(titre));
    }
    
    rechercherParAuteur(auteur) {
        return this.livres.filter(l => l.auteur.includes(auteur));
    }
    
    rechercherParISBN(isbn) {
        return this.livres.find(l => l.ISBN === isbn);
    }
    
    listerLivresDisponibles() {
        return this.livres.filter(l => l.disponible);
    }
    
    listerLivresEmpruntes() {
        return this.livres.filter(l => !l.disponible);
    }
}

Programmation Asynchrone

JavaScript est mono-threadé, mais utilise des mécanismes asynchrones pour gérer les opérations longues sans bloquer l'exécution.

Callbacks

Les fonctions de rappel (callbacks) sont la méthode traditionnelle pour gérer l'asynchronisme.

function faireQuelqueChoseAsync(callback) {
    console.log("Début de l'opération");
    setTimeout(() => {
        console.log("Opération terminée");
        callback(null, "Résultat");
    }, 1000);
}

faireQuelqueChoseAsync((erreur, resultat) => {
    if (erreur) {
        console.error("Erreur:", erreur);
    } else {
        console.log("Succès:", resultat);
    }
});

Problème : L'imbrication excessive de callbacks mène au "callback hell" (enfer des callbacks), rendant le code difficile à lire et maintenir.

Promesses (Promises)

Les promesses offrent une alternative plus propre aux callbacks pour gérer les opérations asynchrones.

function faireQuelqueChoseAsync() {
    return new Promise((resolve, reject) => {
        console.log("Début de l'opération");
        setTimeout(() => {
            const succes = Math.random() > 0.3;
            if (succes) {
                console.log("Opération réussie");
                resolve("Résultat");
            } else {
                console.log("Opération échouée");
                reject(new Error("Quelque chose s'est mal passé"));
            }
        }, 1000);
    });
}

faireQuelqueChoseAsync()
    .then(resultat => {
        console.log("Succès:", resultat);
    })
    .catch(erreur => {
        console.error("Erreur:", erreur.message);
    });

Méthodes Utiles des Promesses

// Promise.all - Attend que toutes les promesses soient résolues
Promise.all([
    fetch('https://api.example.com/data1'),
    fetch('https://api.example.com/data2')
])
.then(responses => {
    console.log("Toutes les données reçues");
})
.catch(erreur => {
    console.error("Une requête a échoué:", erreur);
});

// Promise.race - Retourne la première promesse résolue ou rejetée
Promise.race([
    fetch('https://api.fast.com/data'),
    new Promise((_, reject) => 
        setTimeout(() => reject(new Error("Timeout")), 5000)
])
.then(data => {
    console.log("Données reçues avant timeout");
});

Async/Await

La syntaxe async/await (ES2017) permet d'écrire du code asynchrone de manière synchrone.

async function executerOperations() {
    try {
        console.log("Début");
        
        // Attend la résolution de la promesse
        const resultat1 = await faireQuelqueChoseAsync();
        console.log("Résultat 1:", resultat1);
        
        const resultat2 = await faireQuelqueChoseAsync();
        console.log("Résultat 2:", resultat2);
        
        console.log("Toutes les opérations terminées");
    } catch (erreur) {
        console.error("Erreur:", erreur.message);
    }
}

executerOperations();

Bonnes pratiques : Utilisez toujours try/catch avec async/await pour gérer les erreurs. Une fonction async retourne toujours une promesse.

            throw new Error(`Erreur HTTP: ${reponse.status}`);

        const donnees = await reponse.json();
        console.log(donnees);
    } catch (erreur) {
        console.error("Échec du chargement:", erreur);
    }
}

class GestionnaireTaches {
    constructor() {
        this.taches = JSON.parse(localStorage.getItem('taches')) || [];
        this.filtre = 'toutes';
        
        this.nouvelleTacheInput = document.getElementById('nouvelleTache');
        this.ajouterTacheBtn = document.getElementById('ajouterTache');
        this.listeTaches = document.getElementById('listeTaches');
        this.filtreBtns = document.querySelectorAll('.filtre');
        this.compteTaches = document.getElementById('compteTaches');
        this.supprimerTermineesBtn = document.getElementById('supprimerTerminees');
        
        this.initialiser();
    }
    
    initialiser() {
        this.ajouterTacheBtn.addEventListener('click', () => this.ajouterTache());
        this.nouvelleTacheInput.addEventListener('keypress', (e) => {
            if (e.key === 'Enter') this.ajouterTache();
        });
        
        this.filtreBtns.forEach(btn => {
            btn.addEventListener('click', () => this.changerFiltre(btn.dataset.filtre));
        });
        
        this.supprimerTermineesBtn.addEventListener('click', () => this.supprimerTachesTerminees());
        
        this.afficherTaches();
    }
    
    ajouterTache() {
        const texte = this.nouvelleTacheInput.value.trim();
        if (texte) {
            this.taches.push({
                id: Date.now(),
                texte,
                complete: false
            });
            
            this.sauvegarder();
            this.nouvelleTacheInput.value = '';
            this.afficherTaches();
        }
    }
    
    afficherTaches() {
        // Filtrer les tâches selon le filtre actif
        let tachesFiltrees;
        switch (this.filtre) {
            case 'actives':
                tachesFiltrees = this.taches.filter(t => !t.complete);
                break;
            case 'terminees':
                tachesFiltrees = this.taches.filter(t => t.complete);
                break;
            default:
                tachesFiltrees = this.taches;
        }
        
        // Afficher les tâches
        this.listeTaches.innerHTML = tachesFiltrees
            .map(tache => `
                <li class="${tache.complete ? 'complete' : ''}" data-id="${tache.id}">
                    <input type="checkbox" ${tache.complete ? 'checked' : ''}>
                    <span class="texte-tache">${tache.texte}</span>
                    <span class="supprimer">×</span>
                </li>
            `)
            .join('');
        
        // Mettre à jour le compteur
        const tachesActives = this.taches.filter(t => !t.complete).length;
        this.compteTaches.textContent = `${tachesActives} tâche(s) restante(s)`;
        
        // Ajouter les écouteurs d'événements
        document.querySelectorAll('#listeTaches li input[type="checkbox"]').forEach(checkbox => {
            checkbox.addEventListener('change', (e) => this.basculerComplete(e.target.parentElement.dataset.id));
        });
        
        document.querySelectorAll('#listeTaches li .supprimer').forEach(btn => {
            btn.addEventListener('click', (e) => {
                e.stopPropagation();
                this.supprimerTache(e.target.parentElement.dataset.id);
            });
        });
    }
    
    basculerComplete(id) {
        id = Number(id);
        const tache = this.taches.find(t => t.id === id);
        if (tache) {
            tache.complete = !tache.complete;
            this.sauvegarder();
            this.afficherTaches();
        }
    }
    
    supprimerTache(id) {
        id = Number(id);
        this.taches = this.taches.filter(t => t.id !== id);
        this.sauvegarder();
        this.afficherTaches();
    }
    
    supprimerTachesTerminees() {
        this.taches = this.taches.filter(t => !t.complete);
        this.sauvegarder();
        this.afficherTaches();
    }
    
    changerFiltre(filtre) {
        this.filtre = filtre;
        this.filtreBtns.forEach(btn => {
            btn.classList.toggle('active', btn.dataset.filtre === filtre);
        });
        this.afficherTaches();
    }
    
    sauvegarder() {
        localStorage.setItem('taches', JSON.stringify(this.taches));
    }
}

// Démarrer l'application
new GestionnaireTaches();
